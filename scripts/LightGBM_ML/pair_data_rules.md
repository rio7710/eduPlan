# Pair Data Rules

원문 기반 줄바꿈 복원 학습 데이터 규칙입니다.

## Goal

- 정상 한글 문단을 정답 원문으로 사용
- 원문을 인위적으로 줄쪼개어 입력 쌍 생성
- 라벨은 `keep_break`, `merge_space`, `merge_no_space` 유지

## Source Unit

- 기본 단위는 `문단`
- 제목, 부제, 캡션, 표 셀, 목록 줄은 제외
- 너무 짧은 줄과 숫자 위주 줄은 제외
- HTML 원문이면 본문 텍스트만 추출

## Positive Labels

- `merge_space`
  - 단어와 단어 사이를 줄바꿈으로 끊은 경우
  - 복원 시 공백 1칸을 넣고 합쳐야 함
- `merge_no_space`
  - 단어 내부를 줄바꿈으로 끊은 경우
  - 복원 시 공백 없이 합쳐야 함
- `keep_break`
  - 문단 경계
  - 제목 뒤 본문 시작
  - 목록, 인용, 캡션 시작

## Hard Negatives

- 제목과 본문을 잘못 합친 케이스
- 문단 두 개를 이어붙인 케이스
- 마침표, 물음표, 느낌표 뒤를 억지로 합친 케이스
- 따옴표 종료 뒤 다음 문단을 붙인 케이스
- 기호, 숫자, 불릿으로 시작하는 줄을 본문처럼 붙인 케이스

## Filters

- 좌우 텍스트 길이 2자 미만 제외
- URL, 메일주소, 파일경로 위주 줄 제외
- 한글 비율이 너무 낮은 줄 제외
- 동일 샘플 중복 제거

## Features To Keep

- 좌우 줄 길이
- 좌측 끝문자와 우측 첫문자
- 공백 유무
- 문장부호 종료 여부
- 따옴표, 괄호, 숫자, 불릿 시작 여부
- 제목 패턴 여부
- 한글, 영문, 숫자 비율

## Validation

- 문서 단위 분리로 train/valid 분할
- 같은 문단에서 파생된 샘플은 같은 split 유지
- 정확도보다 `false positive` 감소를 우선 확인
- 제목/부제 오검출은 별도 집계
