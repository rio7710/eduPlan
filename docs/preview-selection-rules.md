# Preview Selection Rules


## Scope

- 대상 화면: `보기` 모드
- 대상 모드: `블록 선택`, `라인 선택`, `문자 선택`
- 기준 문서: Markdown preview HTML 렌더

## Block Split Rules

- 블록 경계는 Markdown 원문의 `---` 로만 판단한다.
- 빈 줄은 블록 경계가 아니다.
- `---` 줄 자체는 선택 콘텐츠에 포함하지 않는다.
- `---` 는 preview 화면에서 가로 divider 로 보여준다.
- 각 블록은 내부적으로 별도 preview block 으로 렌더된다.

## Selection Mode Rules

### 문자 선택

- 브라우저 기본 드래그 선택을 그대로 사용한다.
- 선택 범위는 사용자가 드래그한 텍스트 그대로다.
- 시작점/끝점을 자동 보정하지 않는다.
- 복사는 현재 선택 텍스트 기준으로 처리한다.

### 라인 선택

- 기준은 `보이는 행`이다.
- 드래그 중에는 일반 문자 선택처럼 움직인다.
- 마우스 릴리즈 시 선택 시작점은 첫 행의 맨 앞까지 확장한다.
- 마우스 릴리즈 시 선택 끝점은 마지막 행의 맨 끝까지 확장한다.
- `문단 전체`, `HTML 요소 전체`, `블록 전체`를 선택 기준으로 삼지 않는다.

### 블록 선택

- 기준은 `---` 로 분리된 block range 다.
- 사용자가 드래그 시작한 지점이 속한 블록을 시작 블록으로 본다.
- 사용자가 드래그 종료한 지점이 속한 블록을 끝 블록으로 본다.
- 최종 선택 범위는 시작 블록 콘텐츠 시작부터 끝 블록 콘텐츠 끝까지다.
- `---` 경계 줄은 최종 선택 텍스트에 포함하지 않는다.

## Current Exceptions / Notes

- 현재 preview 는 HTML 렌더 기반이다. CodeMirror 거터 선택과는 다르다.
- 로딩시 기본값으로 사용한다.
- `문자 선택`은 현재 기준 동작으로 본다.
- `라인 선택`은 브라우저 selection API 의 line boundary 동작에 영향을 받을 수 있다.
- heading, list, table, link 등 복합 HTML 구조에서는 `보이는 행` 판정이 브라우저 구현에 따라 달라질 수 있다.
- `블록 선택`은 block wrapper 기준으로 확장한다.
- 블록 구분선은 시각적으로 보여야 하지만, 선택 텍스트에는 포함하지 않는다.

## Non-Goals

- `라인 선택`을 Markdown 원문 line number 기준으로 해석하지 않는다.
- `블록 선택`을 빈 줄 기준으로 나누지 않는다.
- `문자 선택`에 자동 확장 규칙을 넣지 않는다.
